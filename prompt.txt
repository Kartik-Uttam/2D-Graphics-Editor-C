Prompt:
Explain the final 2D Graphics Editor C code.
Response:
The final program is a menu-driven 2D Graphics Editor written in C. It uses a two-dimensional character array to represent a drawing canvas. The user can draw geometric shapes such as lines, rectangles, circles, and triangles using coordinates entered through the keyboard. The program also supports storing shapes as objects, listing objects, deleting objects, and modifying existing objects.
Code:
#define WIDTH 80
#define HEIGHT 24
#define EMPTY '_'
#define PIXEL '*'
#define MAX_OBJECTS 50
char picture[HEIGHT][WIDTH];
Explanation:
The canvas size is fixed as 80 columns and 24 rows. The picture array stores the complete drawing. Initially all positions contain '_', representing empty space. Whenever a shape is drawn, some positions are changed to '*'.
The use of a 2D array makes the program simple because each array element behaves like a pixel on the screen.


Prompt:
Why is the Shape structure used in this project?
Response:
The Shape structure is used to store all information related to a graphical object.
Code:
struct Shape {
int id;
int type;
int x1, y1, x2, y2, x3, y3;
int radius;
};
Explanation:
Every object drawn on the screen needs to store information about itself.
The id field stores a unique identification number.
The type field stores the shape type:
1 = Line
2 = Rectangle
3 = Circle
4 = Triangle
The coordinate fields store the position of the object.
The radius field is used only for circles.
Using structures allows multiple pieces of related information to be grouped together, making object management easier.


Prompt:
Explain clearPicture(), displayPicture() and setPixel().
Response:
These are the foundation functions of the graphics editor.
Code:
void clearPicture()
void displayPicture()
void setPixel(int x, int y)
Explanation:
clearPicture():
This function clears the entire canvas.
Two nested loops visit every location in the picture array and fill it with '_'.
This function is called:
•	At the beginning of the program
•	During redraw operations
•	After clearing all objects
displayPicture():
This function displays the complete drawing.
It prints every row of the picture array so the user can see the current canvas.
setPixel():
This function draws a single pixel.
Before placing '*', it checks whether the coordinate is inside the canvas boundaries.
This prevents invalid memory access and keeps the program safe.
All shape drawing functions eventually use setPixel().


Prompt:
Explain how line, rectangle, circle and triangle are drawn.
Response:
Each shape has a separate function.
Code:
drawLine()
drawRectangle()
drawCircle()
drawTriangle()
Explanation:
drawLine():
This function draws a line between two points.
The algorithm gradually moves from the starting point to the ending point and places '*' characters along the path.
drawRectangle():
A rectangle is drawn using four lines:
•	Top edge
•	Bottom edge
•	Left edge
•	Right edge
Instead of writing rectangle logic separately, the program reuses drawLine().
This demonstrates modular programming.
drawTriangle():
A triangle is created by connecting three vertices.
Three lines are drawn:
•	Point 1 to Point 2
•	Point 2 to Point 3
•	Point 3 to Point 1
drawCircle():
The circle function checks points around the center.
Using the circle equation:
x² + y² = r²
points close to the circumference are plotted using '*'.
This creates an approximate circular shape on the character canvas.


Prompt:
Explain addObject().
Response:
addObject() stores information about every shape drawn by the user.
Explanation:
Whenever a new shape is created:
1.	A unique ID is assigned.
2.	Shape type is stored.
3.	Coordinates are stored.
4.	Radius is stored if the object is a circle.
5.	Object count is increased.
Benefits:
•	Shapes can be identified later.
•	Shapes can be modified.
•	Shapes can be deleted.
•	The picture can be redrawn when required.
Without addObject(), object management would not be possible.


Prompt:
Explain redrawPicture().
Response:
redrawPicture() is one of the most important functions in the project.
Explanation:
Instead of trying to erase only one shape from the canvas, the program follows a simpler approach.
Step 1:
Clear the entire canvas.
Step 2:
Read every object stored in the objects array.
Step 3:
Draw each object again.
This guarantees that the displayed picture always matches the stored object information.
This function is called after:
•	Delete operation
•	Modify operation


Prompt:
Explain deleteObject().
Response:
deleteObject() removes a shape using its ID.
Explanation:
The user enters the object ID.
The function searches through the objects array.
If the object is found:
1.	Remaining objects are shifted one position left.
2.	objectCount is reduced.
3.	redrawPicture() is called.
If the ID does not exist, an error message is displayed.
This approach ensures that the objects array remains continuous without empty gaps.


Prompt:
Explain modifyObject().
Response:
modifyObject() allows the user to change an existing object.
Explanation:
The user enters an object ID.
The program checks the object's type.
For a line or rectangle:
•	New coordinates are taken.
For a circle:
•	New center coordinates
•	New radius
For a triangle:
•	New coordinates for all three vertices
After updating the values:
redrawPicture() is called.
This ensures that the displayed picture immediately reflects the changes.
